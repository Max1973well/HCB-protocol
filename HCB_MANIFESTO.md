## O problema original não é técnico. É humano.

Toda vez que você abre um chat novo com uma IA, ela nasce sem memória.  
Você recomeça. Explica de novo. Reconstrói o contexto.  
Esse "quase" — aquele ajuste fino que consome tempo antes de chegar ao ponto —  
derruba 40% dos tokens úteis e gera ruído, redundância, lixo.

Não é falha da IA.  
É ausência de infraestrutura de continuidade cognitiva.

---

## O que é o HCB

O **Human Context Bus** é o cérebro de silício que carrega as ideias humanas  
e as clarifica de forma que se tornam inteligíveis para qualquer IA.

É o guardanapo do cientista —  
onde a ideia bruta é registrada antes de virar fórmula.

Não é memória da IA.  
É **transporte de contexto humano.**

A IA não "lembra".  
Ela **acorda contextualizada.**

---

## O que o HCB não é

- Não é backup de chat  
- Não é resumo emocional  
- Não é dependente de modelo, versão ou empresa  
- Não é memória infinita — memória infinita gera ruído, não clareza  

---

## O que o HCB é

- Modelo compartilhado de raciocínio  
- Gramática lógica comum entre humano e IA  
- Estado de tensão atual preservado  
- Direção ativa de construção  

---

## Para quem

Para todas as pessoas.  

Especialmente para quem não tem noção do poder de uma IA calibrada.  
Para quem tem limitações cognitivas, fadiga, doenças neurodegenerativas.  
Para quem trabalha sozinho e não pode perder o fio.  

Reduzir a alucinação da IA quase a zero  
significa aumentar a confiança humana,  
diminuir ansiedade  
e permitir melhoria contínua real.

## A arquitetura

```
Alma         → identidade do usuário, valores, limites físicos
HCB Protocol → regras do sistema, estrutura, princípios  
Gaveta       → contexto específico do projeto atual
```

Ordem de carga obrigatória: **Alma → HCB → Projeto**

Cada camada é isolada.  
Nada é carregado automaticamente.  
A herança é sempre por escolha consciente.
